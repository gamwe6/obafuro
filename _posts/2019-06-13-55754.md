<p><strong>対象のテーブル定義と内容</strong></p>

<pre><code>CREATE TABLE r
    (`id` int, `age` int)
;

INSERT INTO r
    (`id`, `age`)
VALUES
    (1, 1),
    (2, 2),
    (3, 5),
    (4, 7),
    (5, 9)
;
</code></pre>

<p><strong>抽出したい結果</strong><br>
「idが3ではない」または「ageが2ではない」レコード</p>

<p>1, 1<br>
4, 7<br>
5, 9</p>

<p><strong>試したSQL1</strong><br>
意図しない結果になりました。<br>
下記はどういう意味になるのですか？</p>

<pre><code> SELECT * FROM `r` WHERE  `id` != 3 OR `age` != 2;
</code></pre>

<p><strong>試したSQL2</strong><br>
意図した結果を取得できたのですが、(ANDなのに)なぜですか？<br>
これは否定の論理和ですか？？</p>

<pre><code>SELECT * FROM `r` WHERE  `id` != 3 AND `age` != 2;
</code></pre>
