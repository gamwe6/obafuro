<p>タイトルの件、IIS上で動作するASP.NETのWEBアプリ(C#)で大量のデータを<br>
SQL Serverから検索して、処理しています。<br>
Webアプリで大量のデータを扱うのは設計が悪いということはわかっておりますが、<br>
件数が増えるにつれて、メモリ領域の確保に時間がかかり検索速度が大幅に遅くなってしまう<br>
状況です。</p>

<p>動作環境は以下となります。<br>
Windows Server 2012 R2<br>
IIS 8.5<br>
メモリ 32GB<br>
SQL Server 2016 Standard Edition<br>
SQL Server も複数インスタンスが同じサーバ上で動作(同居）しており、<br>
リソースモニタで確認すると30GB程度使用中となっています。<br>
IISの利用しているアプリケーションプールの<br>
プライベートメモリ制限と仮想メモリ制限は、0で設定しており、<br>
無制限の認識です。</p>

<p>少しでも状況を改善したいと思っておりますが、単純にメモリを増設すれば<br>
IISが利用できるメモリも増えると考えて問題ないでしょうか？</p>

<p>知見があるかたご回答いただけますと幸いです。</p>
