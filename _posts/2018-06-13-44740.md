<p>Python3.6.3</p>

<p>ディレクトリ構成<br>
sample_project<br>
┣ bin<br>
┃ ┣ __init__.py<br>
┃ ┣ class_BaseModule.py<br>
┃ ┣ class_module.py<br>
┃ ┗ main.py<br>
┣ test<br>
┃ ┣ __init__.py<br>
┃ ┗ test_class_module.py<br>
┗__init__.py</p>

<p>class_BaseModule.py</p>

<pre><code>class base:
    def __init__(self):
        pass
    def get_base(self):
        return 'base'
</code></pre>

<p>class_module.py</p>

<pre><code>import class_BaseModule
class class_sample:
    def __init__(self):
        pass
    def get_(self):
        cls = class_BaseModule.base()
        return cls.get_base()
</code></pre>

<p>main.py</p>

<pre><code>import class_module
cls = class_module.class_sample()
print(cls.get_())
</code></pre>

<p>test_class_module.py</p>

<pre><code>import unittest
import sys
sys.path.append('../')
from bin.class_module import class_sample

class Test_class_module(unittest.TestCase):
    def Test_get_(self):
        cls = class_sample()
        actual = cls.get_()
        self.assertequal('base',actual)

if __name__ == '__main__':
    unittest.main()
</code></pre>

<p>上記の条件でテストすると</p>

<blockquote>
  <p>ModuleNotFoundError: No module named 'class_BaseModule'</p>
</blockquote>

<p>とされてテスト時に読み込んでくれません。</p>

<p>bin\test_class_module.py</p>

<pre><code>import class_module
class Test_class_module(unittest.TestCase):
....
</code></pre>

<p>bin直下にテストモジュールを置く上記条件でテストするとテストが通ります。</p>

<p>テスト時にディレクトリが違う事で宣言したモジュールは読み込みますが明示的に宣言していないモジュールを読まない状態になっていると思います。<br>
今回のディレクトリ構成でclass_BaseModuleモジュールを読む場合ばどのようなコードになりますか？</p>
