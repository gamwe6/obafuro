<p>現在PHPのアプリを作成しています<br>
ほぼ完成し、リリースに向けて調整中なのですが<br>
PHPで発生したスクリプトエラーをIISのInternal Server Errorとして引き渡すことは出来ないでしょうか</p>

<pre><code>&lt;?php
    aa
try{
....
}catch(Exception $ex){
....
}

?&gt;
</code></pre>

<p>としたときに、画面上に</p>

<pre><code>Parse error: syntax error, unexpected 'try' (T_TRY) in C:\inetpub\wwwroot\AAAAA\BBBBB.php on line 3
</code></pre>

<p>と表示されます(例として極端すぎるかもしれませんが)</p>

<p>これを、この様なPHPのエラーでは無く<br>
IISで表示する一般的なエラー画面を表示したいと思っています</p>

<p>PHPのログは</p>

<pre><code>[25-Aug-2015 11:51:25 Asia/Tokyo] PHP Parse error:  syntax error, unexpected 'try' (T_TRY) in C:\inetpub\wwwroot\AAAAA\BBBB.php on line 3
</code></pre>

<p>IISのログは</p>

<pre><code>2015-08-25 02:48:43 POST /AAAAA/BBBBB.php - - 127.0.0.1 Mozilla/5.0+(Windows+NT+6.1;+WOW64;+rv:40.0)+Gecko/20100101+Firefox/40.0 http://localhost/AAAAA/ 200 0 343
</code></pre>

<p>となっており、どうもIIS側は正常終了になってしまっているようです</p>

<p>出来ればIISにもエラーとして通知したいのですがphp.iniの設定などはどこをいじればいいのでしょう</p>
