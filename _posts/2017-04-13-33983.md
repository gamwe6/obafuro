<!-- {% raw %} -->
<p>CentOS7を使っています。</p>

<p>ホストサーバから</p>

<blockquote>
  <p>/mnt/test1 -> 192.168.1.101:/hdd01<br>
  /mnt/test2 -> 192.168.1.102:/hdd01<br>
  /mnt/test3 -> 192.168.1.103:/hdd01</p>
</blockquote>

<p>でNFSマウントをしています。</p>

<p>ホスト側で</p>

<pre><code>$ more /etc/auto.master
/mnt /etc/auto.var    bg,intr --timeout=120 --ghost
$ more /etc/auto.var_dtv
test1 -rw 192.168.1.101:/hdd01
test2 -rw 192.168.1.102:/hdd01
test3 -rw 192.168.1.103:/hdd01
</code></pre>

<p>というautofsの設定を行なっていてマウントはもちろんのことアクセスができるようになっています。</p>

<p>以下のようなdockerを作成しました。</p>

<pre><code>sudo docker run -d --restart=always --privileged --name test -h test -v /mnt/:/mnt/ docker.io/centos /sbin/init
sudo docker exec -it test bash
</code></pre>

<p>でコンテナの中に入り</p>

<pre><code>$ cd /mnt
$ ls
test1 test2 test3
</code></pre>

<p>という状況ですが、</p>

<pre><code>$ cd test1
bash: cd: test1: Too many levels of symbolic links
</code></pre>

<p>というエラーが発生します。</p>

<p>マウント先の192.168.1.101:/hdd01配下にはそれぞれcontentsというディレクトリがあります。</p>

<pre><code>sudo docker run -d --restart=always --privileged --name test -h test -v /mnt/test1/contents/:/mnt/test1/contents/ -v /mnt/test2/contents/:/mnt/test2/contents/ -v /mnt/test3/contents/:/mnt/test3/contents/ docker.io/centos /sbin/init
</code></pre>

<p>とすればコンテナーの中に入って</p>

<pre><code>$ ls /mnt/test1
</code></pre>

<p>で中身を見ることができます。</p>

<p>しかし、今後マウント先が増えることを考えるとその都度dockerを再構成する必要があり、それは避けたいと思っています。<br>
ホスト側でautofsを使ったものが再起動しなくてもコンテナー内で使用（マウント）できるようにするにはどうしたらいいのでしょうか？</p>

<p>ご存知の方、ご教示お願いします。</p>
