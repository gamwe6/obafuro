<p>Rubyの演算子は優先順位が高い順に*、+、>、=なので</p>

```
m=m*3+1&gt;n
```

<p>の場合は</p>

```
m=(m*3+1&gt;n)
```

<p>と同義になるのは納得がいきます。</p>

<p>しかし、(比較演算子からみて)これと左右を逆にした</p>

```
n&lt;m=m*3+1
```

<p>が</p>

```
(n&lt;m)=m*3+1
```

<p>と同義にならず、</p>

```
n&lt;(m=m*3+1)
```

<p>と同義になるのは演算子の優先順位からして違和感を覚えます。<br>
優先順位の他に何か規則があるのでしょうか？</p>
