<p>自作gemをGemfileで指定しinstallを試みた所、installを行うことが出来ず困っています。</p>

<p>GemFile</p>

<pre><code># Gemfile

</code></pre>

<p>上記Gemfileでbundle installを行うと</p>

<pre><code>Could not install to path '/Users/foo/tmp/pkg/gem-name/ruby/2.4.0' because a file already exists at that path. Either remove or rename the file so the directory can be created.
</code></pre>

<p>というエラーが返ってきます。<br>
本来参照していないはずのPathを参照しているようなのですが呼び出し方、或いはrbenv等の設定に問題があるのでしょうか…？<br>
