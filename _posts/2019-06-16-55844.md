<p>お世話になります。</p>

<p>MRubyの拡張機能をc++で作っています。<br>
C++のコードに含まれるMRubyオブジェクトをGCで解放されないようにしたいです。</p>

<p>CRubyのときは</p>

<pre><code>VALUE Data_Wrap_Struct(VALUE klass, RUBY_DATA_FUNC mark, RUBY_DATA_FUNC free, void *sval)
</code></pre>

<p>で引数markにマーク用関数を指定できましたが、MRubyを見ると</p>

<pre><code>#define Data_Wrap_Struct(mrb,klass,type,ptr) mrb_data_object_alloc(mrb,klass,ptr,type)
</code></pre>

<p>引数markがありません。</p>

<p>どこでどうやってマークを付ければ良いでしょうか？</p>

<p>C++側の構造は簡単に書くと以下のようになっています。</p>

<pre><code>class A
{
    mrb_value mrb_object;
};

class B
{
    A* a;
    mrb_value mrb_object;
};

class C
{
    B* b;
    mrb_value mrb_object;
}
</code></pre>

<p>BクラスのaはC++側でmrb_mallocとData_Wrap_Structで作成されます。<br>
更にmrb_gc_mark_value()を呼び出してマークしています。</p>

<p>問題の出るRubyコード</p>

<pre><code>b = B.new
c = C.new
c.b = b

GC.start

p c.b.a
</code></pre>

<p>GC.startでaが解放されてしまい、c.b.aにアクセスするとメモリアクセス例外になってしまいます。</p>

<p>GCで解放されないためにはどうしたらよいでしょうか？</p>
