<p>現在Goを勉強しています。<br>
その過程で他の言語でいうクラスメソッドを作りたいと考えました。<br>
ここで質問なのですが、同じpackageに所属しているクラス(構造体？)同士では、同じ名前のクラスメソッドは作れないのでしょうか？</p>

<p>この質問に至るまでの過程を説明させていただきます。</p>

<p>まずこのクラスメソッドを作りたいと考えた場合、自分の認識ではGoの場合はパッケージにつける形にするしかないと思っています。</p>

```
package models

type User struct {
    Name string
    Gold int
}

func Create() User {
    ~~~
}
```

<p>そしてこれを呼び出すためには、</p>

```
import models

models.Create()
```

<p>こうなりますよね？</p>

<p>しかしこの書き方には問題があります。<br>
なぜなら同じmodelsのパッケージで別のクラス(例えばItemとか)を追加した場合、そのクラスではCreateというメソッド名のクラスメソッドを作ることはできないからです。</p>

<p>私が考えるこの問題の解決方法は２種類あります。<br>
１つはUserCreateみたいにクラス名をメソッドの前につけることです。<br>
２つめはUserとItemなどをそれぞれ別のフォルダに分けて、別のパッケージにしてしまうことです。</p>

<p>ただどちらにも問題があると思っています。<br>
１つめの方法は、UserCreate,ItemCreate,UserAll,ItemAll....というようにmodelsの下に大量でしかも似たようなメソッド名が並ぶことです。</p>

<p>２つめは、その考え方だと一つのクラスに一つのパッケージという事に結果的になることです。<br>
models_user,models_item,....<br>
同じフォルダに複数のパッケージはおけないためフォルダもふえます。</p>

<p>どのようにするのがこの場合良いのでしょうか？<br>
ご回答いただけると助かります。</p>
