<p>RISC-Vの環境下で以下のようなプログラム群をリンクしてから実行したのですが、以下のようなエラーが出て上手く行きません。<br>
エラーの原因が分からないので、改善方法が分かる方がいれば教えて頂けると助かります。<br>
(以下のプログラム内の関数mystrcatの目的はCで書かれたプログラム内で定義された文字列を連結することです。)<br>
mystrcatを呼び出すCのコード(s1の末尾にs2を連結して出力したい。)</p>

<pre><code>#include &lt;stdio.h&gt;
#include &lt;stdlib.h&gt;
extern char* mystrcat(char* s1, char* s2);

int main(){
  char a[100] = "Hello, ";
  char b[100] = "World!";
  printf("%s\n", mystrcat(a,b));
  return 0;
}
</code></pre>

<p>mystrcatを定義したアセンブリコード</p>

<pre><code>    .globl mystrcat
    .align 2

mystrcat:
    #連結される側の文字列のアドレス
    mv s0, a0
    #連結する側の文字列のアドレス
    mv s1, a1
    #関数が扱うデータを保存するためのスタックの確保
    addi sp, sp, -128
    # 各種データのスタックへの退避。
    sd ra, 120(sp)
    sd s0, 112(sp)
    sd s1, 108(sp)

    #mv a0, s0

    jal mystrlen

    #連結される側の文字列の長さ
    mv s2, a0
    sd s2, 100(sp)
    # 関数のための各種データの復元。
    ld ra, 120(sp)
    ld s0, 112(sp)
    ld s1, 108(sp)
    ld s2, 100(sp)

    #s1の文字列のコピーを開始する位置
    add a1, s0, s2
    #コピー元s1のアドレス
    mv a2, s1
loop:
    lb a3, 0(a2)
    beqz a3, end
    sb a3, 0(a1)
    addi a2, a2, 1
    addi a1, a1, 1
    j loop
end:
    sb a3, 0(a1)
    mv a0, s0

    #関数のために確保したスタック領域の解放
    addi sp, sp, 128
    ret
</code></pre>

<p>上のアセンブリコードに必要な補助関数。文字列の長さを返す関数。これにバグはない模様。</p>

<pre><code>    .globl mystrlen
    .align 2

mystrlen:
    #li s0, 0
    #li s1, 0
    #li s2, 0
    li a1,0
calc:   
    lb a2, 0(a0)
    blez a2, end
    addi a0, a0, 1
    addi a1, a1, 1
    j calc
end:
    mv a0, a1
    ret
</code></pre>
