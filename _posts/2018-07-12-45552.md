<p>いつもお世話になります。</p>

<p>標記の件ですが、下記サイトを参考にコーディングしてみました。</p>

<p>Laravelの認証機能でユーザ情報の論理削除（ソフトデリート）を実装する</p>

<p>・・・が、ソフトデリートしたユーザのメールアドレス、パスワードを<br>
ログイン画面で入力するとログインできてしまいます。</p>

<p>そもそも論でお聞きしたいのですが、ソフトデリートしたユーザであっても<br>
ログイン可能なのは仕様なのでしょうか？</p>

<p>ちなみに環境は以下の通りです。</p>

<blockquote>
  <p>Laravel 5.5 <br>
  PHP 7.2.7<br>
  MySQL 5.7.22</p>
</blockquote>

<p>以上、よろしくお願い致します。</p>

<p>※以下、コーディングした内容です。<br>
マイグレーションファイルの作成、マイグレートの実行</p>

<pre><code>$ php artisan make:migration add_column_softDeletes_users_table --table=users
</code></pre>

<blockquote>
  <p>$ view 2018_07_12_045301_add_column_soft_deletes_users_table.php</p>
</blockquote>

<pre><code>&lt;?php

<p>モデルの作成</p>

<pre><code>$ php artisan make:model Models/Users
</code></pre>

<blockquote>
  <p>$ view Users.php</p>
</blockquote>

<pre><code>&lt;?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\SoftDeletes;

class Users extends Model
{
    use SoftDeletes;

    protected $table = 'users';
    protected $dates = ['deleted_at'];
}
</code></pre>

<p>下記SQL文にて、ユーザをソフトデリートしました。</p>

<pre><code>update users set deleted_at = '2018-07-12 01:03:26' where id = 1;
</code></pre>

<p>前項でソフトデリートしたユーザでログインを試みるとログインできてしまいます。</p>
