<p>以下のような構造体があるのですが、<br>
ビッグエンディアン、リトルエンディアン関係なく、<br>
ネットワークバイトオーダーへ変換できる(もしくは型定義できる)コードを書きたいと思っています。<br>
[構造体]</p>

<pre><code>struct sample {
　　unsigned short a:12;
　　unsigned short b:2;
　　unsigned short c:2;
};
</code></pre>

<p>↓これで問題ないですか？</p>

<pre><code>struct sample val = {300,1,2};
htons((uint16_t)val);
</code></pre>

<p>もしくは、以下のように定義すれば、htons()で変換しなくても大丈夫でしょうか？</p>

<pre><code>struct sample {
#if __BYTE_ORDER == __LITTLE_ENDIAN
　　unsigned short a1:8;　/* aの前半 */
　　unsigned short c:2;
　　unsigned short b:2;
　　unsigned short a2:4;　/* aの後半 */
#endif
#if __BYTE_ORDER == __BIG_ENDIAN
　　/* ビッグエンディアンならそのまま流してOK？ */
　　unsigned short a:12;
　　unsigned short b:2;
　　unsigned short c:2;
#endif
};
</code></pre>
