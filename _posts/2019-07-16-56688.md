<p>GKEを用いてWordpress上にサイトを構築しています。<br>
ローカル環境で動作するコンテナをデプロイし、ロードバランサとしてservice起動しました。</p>

<p>サイトへのアクセスはできるのですが、相対パスで記述した静的ファイル(cssやjs,hrefで指定した遷移先パスなど)が読み込めず、原因を調査しています。</p>

<p>サービス起動直後は静的ファイルが読み込めるため、loadbalancerによるスケーリングが原因と考えています。</p>

<p>静的ファイルを読み込むためのyamlファイルの記述方法をご教示いただけないでしょうか。</p>

<p><a href="https://i.stack.imgur.com/S6dMk.png" rel="nofollow noreferrer"><img src="https://i.stack.imgur.com/S6dMk.png" alt="コンソールエラー画面"></a></p>

<p>service.yamlファイル</p>

<pre><code>labels:

apiVersion: v1
kind: Service
metadata:
  labels:
    app: wordpress
  name: wordpress
spec:
  type: LoadBalancer
  loadBalancerIP: 34.85.72.79
  ports:
    - port: 80
      targetPort: 80
      protocol: TCP
  selector:
    app: wordpress
</code></pre>

<p>deployment.yaml</p>

<pre><code>apiVersion: apps/v1
kind: Deployment
metadata:
  name: wordpress
  labels:
    app: wordpress
spec:
  replicas: 1
  selector:
    matchLabels:
      app: wordpress
  template:
    metadata:
      labels:
        app: wordpress
    spec:
      containers:
        - image: gcr.io/ganalytics-1312/oohmatch_wordpress
          name: wordpress
          env:
          - name: WORDPRESS_DB_HOST
            value: mysql:3306
          - name: WORDPRESS_DB_PASSWORD
            valueFrom:
              secretKeyRef:
                name: mysql
                key: password
          ports:
            - containerPort: 80
              name: wordpress
          volumeMounts:
            - name: wordpress-persistent-storage
              mountPath: /var/www/html
      volumes:
        - name: wordpress-persistent-storage
          gcePersistentDisk:
            pdName: wordpress-disk
            fsType: ext4
</code></pre>
