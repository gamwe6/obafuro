<p>手元のプロジェクトでいろんなツールをインストールしていった結果、時折、次のようなエラーメッセージが発生するようになりました。</p>

<pre><code>The `infocmp' command exists in these Python versions:
  anaconda3-5.1.0
</code></pre>

<p>これは、自分の理解ですと、 <code>infocmp</code> のプログラムファイルが、 <code>anaconda3-5.1.0</code> の python 上ではインストールされているけれども、今デフォルトで使っている python のバージョン(<code>3.6.5</code>)にはインストールされていないがために発生していると思っています。</p>

<p>これを解消するために、この <code>infocmp</code> をインストールしたいのですが、このコマンドが何のパッケージからインストールされたものなのか調べる方法がわからずにいます。</p>

<h3>質問</h3>

<ul>
<li>pip によってインストールされたコマンドが、どのパッケージ由来のものであるか、調べる方法はありますでしょうか。</li>
</ul>

<h3>環境</h3>

<p>pyenv の上で動かしており、今現在は <code>3.6.5</code> のバージョンをデフォルトで使いたいと思っています。</p>
