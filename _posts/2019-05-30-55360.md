<p>Gitでプロジェクトを管理していたのですがGitがおかしくなってしまいました。<br>
おかしくなった原因は明確で、<br>
今まで「ファイル名の大文字小文字を区別しない」というGitの設定でコミットなどをしていたのに<br>
途中から「ファイル名の大文字小文字を区別する」という設定に変えたからです。<br>
↓のコマンドにて</p>

```
git config core.ignorecase false
```

<p><strong>【この設定に変えたかった理由は】</strong><br>
パッケージ名の大文字小文字を変更<br>
Javaプロジェクトなので自動的にディレクトリ名も大文字小文字が変わる<br>
ソースないの　package　部分も自動リファクタリングにより大文字小文字が変わる<br>
↓<br>
コミット<br>
↓<br>
パッケージ名を変える前の過去のコミットに戻す<br>
↓<br>
ソース内の　package　部分は大文字小文字変更前に戻る<br>
ディレクトリは大文字小文字が変更前に変わらず、変更後の状態のまま</p>

<p>となるからです。</p>

<p>途中から大文字小文字を区別するようにするにはどうすればよろしいでしょうか？<br>
ちなみに、SourceTreeを使用しており、リモートリポジトリもあります。</p>

<hr>

<h2>以下追記</h2>

<p>【要点まとめ】<br>
・masterブランチとBブランチがあるとする<br>
・どちらも「ファイル名の大文字小文字は区別しない」として今までコミットされてきた<br>
・先行するmasterブランチで作業中に「ファイル名の大文字小文字も区別されないと困る」事が発覚</p>

<p>【現状】（ファイル名もディレクトリ名も同じ）<br>
・masterブランチでとあるファイルのファイル名の大文字小文字を変更<br>
（例　aaa/bbb.png　→　AAA/bbb.png）<br>
・Bブランチに切り替えてみると<br>
過去の状態（aaa/bbb.png）であるはずのBブランチに切り替えても<br>
ファイル名が「AAA/bbb.png」になってしまう</p>

<p>【理想】<br>
Bブランチに切り替えた時、Bブランチは「aaa/bbb.png」の状態だったので「aaa/bbb.png」にファイル名が戻って欲しい。</p>
