<p>有効フラグ、削除フラグなどのフラグ項目に NOT NULL 制約を付与しないDB設計をよく見かけます。「仕様上は 0 と NULL は同じ意味で扱ってください」と指示されるのですが、NOT NULL 制約（とデフォルト値）を付与することに何らかのデメリットがあるのでしょうか。</p>

<p>フラグ項目に限らず一般的に、 NULL を格納する必要のない項目に NOT NULL 制約を付与することで生じるデメリットは何かあるでしょうか。</p>
