<p>いつもお世話になっております。</p>

<p>httpdのアクセスログを、fluentdで件数をカウントしたいと考えております。<br>
fluent-plugin-datacounterで実現できると思ったのですが、<br>
fluentdを起動した時に以下のエラーロクが表示され起動されませんでした。<br>
解決手段が分からなくご教授頂けると幸いです。</p>

<p>エラー内容</p>

<pre><code>2017-02-28 19:27:20 +0900 [warn]: both of Plugin @id and path for &lt;storage&gt; are not specified. Using on-memory store.
2017-02-28 19:27:20 +0900 [error]: dry run failed: undefined method `workers' for #&lt;Fluent::SystemConfig:0x007fade2d53620&gt;
</code></pre>

<p>環境</p>

<pre><code># cat /etc/redhat-release
CentOS release 6.8 (Final)

# td-agent --version
td-agent 0.14.11

# ruby --version
ruby 2.4.0p0 (2016-12-24 revision 57164) [x86_64-linux]

# td-agent-gem list --local | grep fluent-plugin-datacounter
fluent-plugin-datacounter (1.0.0)
</code></pre>

<p>httpdのアクセスログ形式(tlsv形式)</p>

<pre><code>2017-02-28 19:25:13.000000000 +0900 httpd.access: {"forwardedfor":"-","host":"127.0.0.1","req":"OPTIONS * HTTP/1.0","status":"200","size":"0","referer":"-","ua":"Apache (internal dummy connection)","reqtime_microsec":"284","cache":"-","runtime":"-","vhost":"-"}
</code></pre>

<p>設定内容</p>

<pre><code># vi /etc/td-agent/td-agent.d/httpd_count.conf
&lt;source&gt;
  @type tail
  path /var/log/httpd/access_log
  pos_file /var/log/td-agent/pos/access_log.pos
  tag httpd.access
  format ltsv
  time_key     time
  time_format  %Y-%m-%dT%H:%M:%S%z
&lt;/source&gt;
&lt;match httpd.**&gt;
  @type datacounter
  count_interval 1m
  count_key status
  pattern1 2xx ^2\d\d$
&lt;/match&gt;
&lt;match **&gt;
  @type stdout
&lt;/match&gt;
</code></pre>

<p>お手数ですがよろしくお願い致します。</p>
