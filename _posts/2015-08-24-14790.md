<p>↓のような処理では「エラーが発生しない処理」もtry-catchを囲むべきでしょうか？<br>
エラーが発生しない処理の長さも勘案して決めるものなのでしょうか？<br>
お手数をおかけしますが、教えて頂けると幸いです。<br>
今回は実行不可能系のエラーは無視して考えています。<br>
よろしくお願いします。</p>

<pre><code>/* エラーが発生するかもしれない処理1 */
/* エラーが発生しない処理 */
/* エラーが発生するかもしれない処理2 */
</code></pre>

<p>　<br>
１．エラーが発生しない処理もtry-catchに囲む</p>

<pre><code>try {
    /* エラーが発生するかもしれない処理1 */
    /* エラーが発生しない処理 */
    /* エラーが発生するかもしれない処理2 */
} catch (Exception1 $e){
} catch (Exception2 $e){
}
</code></pre>

<p>２．エラーが発生しない処理はtry-catchに囲まない</p>

<pre><code>try {
    /* エラーが発生するかもしれない処理1 */
} catch (Exception1 $e){
}
/* エラーが発生しない処理 */
try {
    /* エラーが発生するかもしれない処理2 */
} catch (Exception2 $e){
}
</code></pre>
